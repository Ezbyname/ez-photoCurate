# E-z Photo Organizer

An intelligent photo curation pipeline for building meaningful image collections for special events — bar/bat mitzvas, weddings, birthdays, photo books, and more.

Scans multiple image sources, deduplicates, detects faces, categorizes by date/age, and uses an **event-aware agent** to auto-select the best diverse images and generate an interactive HTML gallery for review.

## Features

- **Event templates** — pre-built category structures for bar mitzvas, weddings, birthdays, vacations, photo books, baby's first year
- **Multi-source scanning** — USB drives, cloud exports (Google Takeout), local folders, OneDrive
- **Smart deduplication** — hash-based + visual similarity detection
- **Face recognition** — identify specific people using reference photos (dlib-based)
- **Auto-categorization** — assign images to age brackets, time periods, or custom categories
- **Event agent** — analyzes your collection, identifies gaps, and auto-selects the best images per category
- **Interactive HTML gallery** — review thousands of images with filters, move between categories, reject bad ones
- **Incremental scanning** — re-scan is fast, only processes new images
- **Quality gates** — auto-skip tiny images, low-res, screenshots

## Quick Start

```bash
# 1. Install dependencies
pip install pillow numpy face_recognition

# 2. See available event templates
python curate.py templates

# 3. Create config from a template
python curate.py init --event bar_mitzva --birthday 2013-07-16
# or
python curate.py init --event wedding --event-date 2025-06-15

# 4. Edit curate_config.json — add your image source paths

# 5. Scan all sources
python curate.py scan --config curate_config.json

# 6. Analyze your collection
python curate.py analyze

# 7. Auto-select the best images per category
python curate.py auto-select --strategy balanced

# 8. Review in interactive gallery
python curate.py report

# 9. Export final collection
python curate.py apply curate_changes.json --output ./final_collection
```

## Event Templates

Choose a template to get a pre-built category structure tailored to your event:

| Template | Categories | How it categorizes |
|----------|-----------|-------------------|
| `bar_mitzva` | 13 age brackets (birth → 13) | Child's birthday + photo date |
| `wedding` | 13 (prep, ceremony, party, portraits...) | Event date + time of day |
| `birthday` | 8 (setup, cake, games, candids...) | Event date + time of day |
| `baby_first_year` | 15 monthly milestones | Baby's birthday + photo date |
| `photo_book` | 10 (seasons + themes) | Year + month |
| `vacation` | 12 (per-day + themes) | Trip start date + day offset |

```bash
python curate.py templates                                          # list all
python curate.py init --event bar_mitzva --birthday 2013-07-16      # age-based
python curate.py init --event wedding --event-date 2025-06-15       # time-based
python curate.py init --event photo_book --year 2024                # season-based
python curate.py init --event vacation --event-date 2025-08-01 --end-date 2025-08-07
```

## Configuration

`curate_config.json` (generated by `init`):
```json
{
  "event_type": "bar_mitzva",
  "subject_birthday": "2013-07-16",
  "ref_faces_dir": "./ref_faces",
  "face_names": ["reef"],
  "face_tolerance": 0.6,
  "sources": [
    {"path": "D:\\photos", "label": "USB Drive"},
    {"path": "C:\\Users\\me\\Pictures", "label": "Local Pictures"},
    {"path": "C:\\takeout\\Google Photos", "label": "Google Takeout"}
  ],
  "target_per_category": 75,
  "min_size_kb": 80,
  "min_dim": 600,
  "thumb_size": 120
}
```

### Face Recognition Setup

Create reference face directories with clear photos of each person:
```

  another_person/
    photo1.jpg

## Event Agent

The event agent (`event_agent.py`) provides intelligent curation based on event-specific knowledge:

### `analyze` — Understand your collection
```
python curate.py analyze
```
- Per-category status: EMPTY / LOW / CLOSE / OK / EXCESS
- Source diversity analysis
- Face detection coverage
- Actionable recommendations (critical gaps, overflow, tips)
- Event-specific priorities (e.g., "ceremony should be ~15% of a wedding collection")

### `auto-select` — Pick the best images automatically
```
python curate.py auto-select --strategy balanced
```
Three selection strategies:
- **`balanced`** (default) — quality-ranked with diversity filtering at 0.85 threshold
- **`quality`** — prioritize resolution and faces, loose diversity (0.95 threshold)
- **`diverse`** — maximize visual variety, strict diversity (0.75 threshold)

Quality scoring is event-aware:
- Bar mitzva: target face 3x weight, any face 1.5x, resolution 1x
- Wedding: resolution 1.5x weight, target face 2x, any face 1x
- Birthday: target face 2.5x, candids and group shots prioritized

### Event Knowledge Base

Each event type includes:
- **Total target range** — e.g., wedding 300-500, birthday 100-200
- **Balance rules** — "don't overload baby years", "ceremony is the core"
- **Quality weight profiles** — what matters most for this event
- **Curation priorities** — what makes a great collection

## Interactive Gallery

The HTML report provides:

- **Thumbnails** for all images grouped by category
- **Filters** — by source, face detection status, qualification status, filename search
- **Selection** — click to select, shift+click for range, select/deselect all per category
- **Move** — promote pool images into categories or reject bad ones to pool
- **Undo** — revert last action
- **Export** — save changes as JSON for applying to filesystem
- **Lightbox** — double-click any image for full-size preview with metadata

                              v

                                        |
                              +---------+---------+
                              |                   |
                              v                   v
                    curate.py analyze    curate.py auto-select
                              |                   |
                              v                   v
                     recommendations        best N per category
                              |                   |
                              +---------+---------+
                                        |
                                        v
                              curate.py report --> curate_report.html
                                        |
                                        v
                              curate.py apply --> final_collection/
```
